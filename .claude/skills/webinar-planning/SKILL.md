# SHIFTAI ウェビナー企画立案スキル

## このSkillが解決する「私たちのチームの」課題

- 「ウェビナーのテーマは思いついたけど、応募フォームの各項目を埋めるのが面倒」
- 「ターゲットやゴール設定が甘く、ありきたりな内容になってしまう」
- 「タイトルが弱く、集客に結びつかない」
といった課題を、AIを壁打ち相手にすることで素早く解決し、そのまま提出できるデータを生成します。

## 使い方

`npx skills ウェビナー企画` またはこのプロンプトを読み込ませることで起動します。
起動後、AIが以下のステップに従ってあなたに質問を投げかけますので、対話ベースで進めてください。

## 実行フロー

AIは以下のステップに沿って進行し、決して一度に全てを完了させず、対話をしながら企画をブラッシュアップしてください。

### STEP 1: アプローチの選択（企画の種出し）
まずは、どの方法でウェビナーのテーマを決めるか、ユーザーに以下の4つの選択肢を提示し選ばせてください。

1. **社内・PC上の知見から抽出モード**
   - まずAIからユーザーに対し、「**PC内のファイル（過去のプロジェクト資料、スクリプト、AI活用メモなど）を検索・読み込みしてもよろしいですか？**」と許可を求めます。
   - ユーザーから「OK」が出た場合のみ、AIが自律的にPC内の情報を漁り、ユーザーが過去にAIを活用した事例等の「既存ノウハウ」を見つけ出し、そこからウェビナー化できそうな実践的なテーマを3つ抽出して提案する。
2. **AIからの逆提案モード**
   - 最近のAI系ニュースやトレンド、ビジネスでの需要をもとに、あなたが「今やるべきテーマ案」を3つ提案し、ユーザーが選ぶ。
3. **持ち込み企画モード**
   - ユーザーがすでに考えている「こんな企画やりたい」というアイデアをベースに、あなたがヒアリングを開始する。
4. **お悩み解決モード**
   - 最近、受講生やクライアントからよく聞かれる具体的な「悩み」や「トラブル」を1つユーザーが挙げ、そこからウェビナーのテーマをあなたが逆算して提案する。

ユーザーが選択肢（1〜4）を選んだら、そのモードに従ってコアとなる「テーマ」「レベル（1:初心者〜4:トップレベル）」「企画背景」を決定し、次のSTEPへ進んでください。

### STEP 2: ターゲットと課題の深掘り
STEP 1で決まった内容をもとに、以下の人物像の仮説をユーザーに提示し、すり合わせてください。
- ターゲットの詳細なイメージ（年齢、職業など）
- そのターゲットが持つリアルな「悩みや課題」

### STEP 3: 多角的な検証とブラッシュアップ
ターゲットとゴールが仮決定したら、ユーザーに「こちらの内容で進めてよいか？」を確認してください。
ユーザーがOKを出したら、以下の3つの観点で企画内容を厳しく検証し、結果と修正案を提示してください。

1. **マネタイズ・ビジネス貢献の確認**
   - この企画はマネタイズに繋がるか？または、既存のマネタイズ（仕事への応用）事例に関連づけられるか？
2. **ターゲットへの「刺さり具合」の確認**
   - このターゲット層が「絶対参加したい！」と熱量を持つ内容か？
3. **3つの思考法による多角レビュー**
   - **ロジカルシンキング**: 構成に矛盾はないか、初心者が順序立てて実践できるか。
   - **クリティカルシンキング**: 「そもそもこのツールを使う必要があるか？」「この前提は正しいか？」などの懐疑的な視点。
   - **ラテラルシンキング**: 「別の切り口はないか？」「全く違う業界の事例を混ぜたら面白くならないか？」といった突飛なアイデア。

### STEP 4: タイトル案の提示と最終確認
STEP 3の多角レビューが終わって内容が固まったら、最終的な「タイトル案」を3つ提示し、ユーザーに最終確認を取ってください。
※タイトルの条件：**単なる説明文ではなく、ターゲット層に深く刺さり、強烈に印象に残る（キャッチーでベネフィットが明確な）言葉選び**をすること。

### STEP 5: 最終出力（PDF化用データ ＆ 応募フォーム用データ）
ユーザーが最終確認でOKを出したら、以下の2つのフォーマットで出力して完了としてください。

#### 【出力1：詳細企画書のPDFファイル生成】
※あなた（AIエージェント）が Antigravity や Claude Code 等のローカル実行環境（ターミナル操作）を持っている前提の指示です。
ターミナルのコマンド（Pandocやnpmパッケージ）やスクリプト言語等を利用して、Step3までで決まった全ての内容を含む綺麗なPDFファイル（ファイル名例：`ウェビナー企画書_タイトル.pdf`）をローカル環境に直接ファイルとして生成・保存し、そのファイルのフルパスをユーザーに提示してください。

#### 【出力2：応募フォーム提出用データ】
※こちらは指定の応募フォームにコピペしやすいよう、**「見出し」の下の「回答内容」だけをそれぞれ個別のコードブロックにして**出力してください。

**タイトル案**
```text
[決定したタイトル]
```

**カテゴリ**
```text
[AIエンジニア / 画像生成・動画生成 / AIトレンド / マネタイズ / ツール解説 / AIコンサルタント・AI講師 / ライティング・文章生成 / 業務効率化 / 社内推進 / その他 から最も近いものを1つ]
```

**ウェビナーレベル**
```text
[1〜4の数字と説明]
```

**企画背景**
```text
[企画の背景（会員の成果や実績が出る視点、相談内容など）]
```

**ターゲット**
```text
[参加して欲しい人の具体的イメージ]
```

**ターゲットが持つ悩みや課題**
```text
[ターゲットが抱える具体的な悩み・課題]
```

**ベネフィット**
```text
[参加することで得られる利点]
```

**参加者のゴール**
```text
[ウェビナーで何を作れるか・持ち帰れるか]
```

**アジェンダイメージ**
```text
[ウェビナーの構成案]
```

**担当モデレータ（希望あれば）**
```text
[特になければ「未定」]
```

**その他**
```text
[参考リンクなどがあれば記載]
```

## 特に重要なポイント（勘所）
- **課題:** AIが一般的なアイデアしか出してこない場合。
- **解決策/コツ:** STEP1や2の回答時に、「最近Discordでこういう質問が多かった」など、リアルな一次情報を混ぜて回答すると、より実践的なアジェンダが生成されます。

## このSkillがチームに与えるインパクト
応募フォームに合わせた形で出力されるため、企画の壁打ちから提出までの工数が大幅に削減され、コンテンツの作り込みに時間を割けるようになります。また、3つの思考法レビューにより企画の質が格段に向上します。
