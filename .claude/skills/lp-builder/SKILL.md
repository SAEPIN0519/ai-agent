# lp-builder

## ワークフロー

- [ ] **Phase 1-A: ヒアリング（基本情報）** — AskUserQuestion × 1回で4問を一度に聞く
  - ページの種類（LP / ポートフォリオ / サービス紹介 / イベント / 採用 / その他）
  - ターゲット（誰に届けたいか）
  - 提供するもの（サービス名・商品名・内容 + 強み・差別化ポイント）
  - CTA（最終的に何をしてほしいか。例：問い合わせ・購入・申し込み・フォロー）

- [ ] **Phase 1-B: ヒアリング（ビジュアル）** — AskUserQuestion × 1回で3問を一度に聞く
  - トーン（モダン / 温かみ / クール / エレガント / ポップ）
  - カラーイメージ（あれば。例：「深いネイビー×ゴールド」「パステルピンク系」）
  - 参考にしたいサイト・世界観（なくてもOK）

- [ ] **Phase 1-C: 画像ヒアリング** — 以下の分岐でヒアリングする

  **【参考サイトが提示された場合】**
  - WebFetch で参照サイトにアクセスし、ビジュアル構成を分析する
  - 分析結果をユーザーに提示する（例：「このサイトはヒーロー全画面背景 + セクションごとに斜め背景を使っています」）
  - 再現に必要な素材・要素をユーザーに伝える：
    - 画像が必要な箇所（例：「ヒーロー背景画像1枚、Aboutセクション人物写真1枚」）
    - ロゴ・アイコン・フォントなどのアセット
    - 特殊なUIパターン（アニメーション、パララックス、動画背景など）
  - 各画像について「お手持ちの画像を使いますか？それとも生成しますか？」を確認
  - 生成する場合 → 後述のプロンプト提示フローへ

  **【参考サイトなし・画像ありの場合】**
  - 「使いたい画像は何枚ありますか？どのセクションに使いますか？」と確認する
  - ヒーロー背景 / セクション背景 / コンテンツ内（人物・製品・イラスト）を区別する
  - 使用箇所が決まったら各画像の意図・雰囲気を確認する

  **【画像なしの場合】**
  - CSS のみで構成する（グラデーション背景、アニメーションblob、SVGウェーブ）
  - このLP（Claude&nbsp;Code 勉強会）のように、カラーパレットで統一感を出す

  **【画像を生成したい場合 or 追加したほうがよいと判断した場合】**
  - 必要な画像の種類・枚数・配置を提案する（例：「ヒーロー背景1枚＋Aboutセクション1枚を推奨」）
  - 各画像について Gemini / niji・journey / Midjourney 向けのプロンプトをそれぞれ提示する
  - プロンプトには必ず以下を含める：
    - 用途（hero background / section background / product image など）
    - 色調（LPのカラーパレットに合わせる）
    - アスペクト比（ヒーロー背景なら16:9、ポートレートなら3:4など）
    - 「No text, no letters, no words」（文字なし指定）
    - スタイル（photorealistic / illustration / abstract など）
  - 「画像が準備できたらファイル名を教えてください。LPに組み込みます。」と伝える

- [ ] **Phase 2: 構成・コピー設計（提案 → 確認）** — ユーザーにOKをもらってからPhase 3へ
  - `references/section-templates.md` のページ種別テンプレートを参照して7〜9セクション構成を提案
  - ヘッドラインコピー案を3パターン提示（`references/copy-formulas.md` の公式を活用）
  - 各セクションのメッセージ軸を箇条書きで提示
  - 「この構成・コピーでOKですか？修正があれば教えてください。」と確認する

- [ ] **Phase 3: HTML生成** — 1ファイルに `<style>` 埋め込みで完結
  - `references/design-rules.md` のコンマリデザイン原則を全適用する
  - `references/color-palettes.md` からトーン・ジャンルに合ったパレットを選び、CSS変数で定義
  - 必須要素（下記）をすべて含める：
    - `<style>` 埋め込み（CSS変数でカラー管理）
    - Material Icons（`https://fonts.google.com/icon?family=Material+Icons`）を使用、絵文字禁止
    - Google Fonts でフォント指定（Noto Sans JP 推奨）
    - スクロールアニメーション（Intersection Observer + `.fi` クラス）
    - レスポンシブ（モバイルファースト、`@media (max-width: 768px)`）
    - スクロールプログレスバー
    - ナビゲーション（スクロール時に背景変化）
    - ウェーブ区切り（セクション間のSVGウェーブ）
    - フッター
  - 絶対にやらないこと：絵文字の使用、情報商材っぽいデザイン、過度な装飾

- [ ] **Phase 4: プレビュー & 確認**
  - `.claude/launch.json` が存在しない場合は作成してプレビューサーバーを起動
  - `preview_screenshot` で表示確認（全体 + スクロール箇所）
  - チェック項目（必ず全項目目視確認）：
    - テキストはみ出し
    - **日本語改行の不自然な折り返し**（下記の典型バグを必ずチェック）
      - `<h1>` `<h2>` などの大見出しで、意味の切れ目ではない場所で折り返されていないか（例：「時代が」と「来た。」が別行になるなど）
      - 大見出しには `word-break: keep-all` を必ず付与し、カナ/漢字の途中での折り返しを防ぐ
      - `<p>` 要素内をHTML上で複数行に書くと、改行がスペースに変換されて文中に不自然な空白が入る → `<p>` 内のテキストは1行で書くか、明示的に `<br>` を使う
      - `display: block` を持つ `<span>` 内の長いテキストは画面幅次第で途中折り返しが起きる → `word-break: keep-all` + `overflow-wrap: anywhere` を設定する
    - グリッド崩れ
    - モバイル表示（幅375px）での折り返し
    - プレースホルダーリンク残存（`href="#"`）
  - 「表示確認しました。修正点があれば教えてください。」と伝える

- [ ] **Phase 5: 保存**
  - 「どのディレクトリに保存しますか？」とユーザーに確認する
  - 指定パスに `index.html` として保存
  - 保存後、ファイルパスをユーザーに伝える
