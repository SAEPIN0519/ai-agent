## AIDMA {#aidma}

**構造：** Attention → Interest → Desire → Memory → Action

最も基本的なLP構成に対応する。

| フェーズ | セクション | コピーの焦点 |
|---------|-----------|------------|
| Attention | Hero | 「あなたのことを言ってる」と感じさせる一言 |
| Interest | Problem / About | 共感・悩みの言語化 |
| Desire | Features / Results | 「それが欲しい」と思わせる価値提示 |
| Memory | Testimonials | 「自分と同じ人が成功した」証拠 |
| Action | CTA | 明確で簡単な次の行動 |

## PAS / PASO {#pas}

**PAS構造：** Problem → Agitation → Solution

**PASO構造：** Problem → Agitation → Solution → Outcome

```
【Problem】
〇〇に悩んでいませんか？

【Agitation】
このまま続けると、〇〇になってしまうかもしれません。
実際に〇〇という声をよく聞きます。

【Solution】
そこで〇〇（サービス名）が生まれました。

【Outcome（PASへの追加）】
〇〇を使った方からは「〇〇が変わった」という声が届いています。
```

**使いどころ：** 悩みが明確なターゲット向けLP、問題意識が高い層へのアプローチ

## BEFORE-AFTER-BRIDGE {#bab}

**構造：** Before（現状の苦しさ） → After（理想の姿） → Bridge（その橋渡し）

```
【Before】
毎日〇〇に追われて、本当にやりたいことができていない。

【After】
〇〇が自動化されたら、あなたの時間はどう変わるでしょうか。

【Bridge】
〇〇（サービス名）は、〇〇を使って〇〇を実現します。
```

**使いどころ：** 変化・成長・ビフォーアフターが見えやすいサービス

## 4U原則（ヘッドライン用）{#4u}

**4U：** Useful（有益）× Urgent（緊急性）× Unique（独自性）× Ultra-specific（超具体的）

```
× NG: 「生産性が上がるツールです」
○ OK: 「週3時間のルーティン作業を10分にする、AIメール整理ツール」
       ↑有益    ↑超具体的           ↑独自性
```

**ヘッドライン生成の手順：**
1. Who（誰が）× What（何を）× How（どうなる）を書き出す
2. 数字・期間・固有名詞で具体化する
3. 4Uのうち2〜3つが入っているか確認する

## FAB（特徴→利点→便益）{#fab}

**構造：** Feature（特徴）→ Advantage（利点）→ Benefit（便益）

```
【Feature】この機能は〇〇です
     ↓
【Advantage】だから〇〇ができます
     ↓
【Benefit】つまり、あなたは〇〇できるようになります
```

**例：**
- Feature: 月額定額プラン
- Advantage: 使い放題で追加費用なし
- Benefit: 予算を気にせず使い続けられる

**使いどころ：** Features セクションのカード文章、サービス説明

## ヘッドライン型別テンプレート {#headline-templates}

### How-to型
```
〇〇をしなくても、〇〇できるようになる方法
〇〇なのに、〇〇が手に入る理由
```

### 数字型
```
〇〇人が選んだ、〇〇の方法
たった〇〇日で、〇〇を実現した
〇〇%が「〇〇だった」と答えた理由
```

### 問いかけ型
```
〇〇で悩んでいませんか？
まだ〇〇を使っていないのですか？
```

### 約束型
```
〇〇を保証します
〇〇ができなければ全額返金
```

### ターゲット特定型
```
〇〇のあなたへ
〇〇に悩む〇〇のために作りました
```

## セクション別コピー設計 {#section-copy}

### Hero セクション

| 要素 | 文字数目安 | 役割 |
|------|-----------|------|
| eyebrow（上部小見出し） | 10〜20字 | ジャンル・ターゲット明示 |
| ヘッドライン（h1） | 20〜35字 | 最も伝えたい一言（4U原則） |
| サブコピー（p） | 40〜80字 | ヘッドラインを補足・具体化 |
| CTAボタン | 8〜15字 | 具体的な行動（「〇〇を始める」） |

```html
<!-- 構造例 -->
<span class="hero-eyebrow">AIデザイン × フリーランス向け</span>
<h1>デザインの悩みを、<span class="accent">AIで解決する。</span></h1>
<p>Canva・Midjourney・ChatGPTを組み合わせた新しいデザインフローを<br>
   一緒に学びながら、実践ですぐ使えるスキルを身につけよう。</p>
<a href="#contact" class="btn-primary">
  <span class="material-icons">arrow_forward</span>
  無料相談を予約する
</a>
```

### Problem セクション

```
「〇〇しているのに、なかなか〇〇できない」
「〇〇に時間がかかりすぎて、〇〇に集中できない」
「〇〇を試したけど、結局〇〇のまま」

そんな経験、ありませんか？
```

### Features セクション

各カード = FAB構造（Feature + Benefit）

```
アイコン
機能名（10字以内）
──
この機能で〇〇ができます。
だから〇〇に時間を使えるようになります。（2〜3行）
```

### Testimonials セクション

```
「〇〇でした。でも〇〇を使ったら、〇〇になりました。
 特に〇〇が役立っています。」

── 〇〇 様（属性：職業・セミナー名等）
```

### CTA セクション（最終）

```
ヘッドライン: 「今すぐ始めましょう」より「〇〇のための第一歩を」
サブコピー: ハードルを下げる言葉「まずは無料で〇〇できます」
注記: 「クレジットカード不要」「いつでも解約可能」
```
