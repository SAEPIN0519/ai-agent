# Power BI ダッシュボード構築手順

## 全体構成

```
┌─────────────────────────────────────────────────────────┐
│  【技術】更新型進捗管理ダッシュボード                          │
├──────────────────────┬──────────────────────────────────┤
│  ① 担当別案件数       │  ② ステータス別積み上げ棒グラフ       │
│  （円グラフ）          │  （担当×ステータス）                │
├──────────────────────┼──────────────────────────────────┤
│  ③ KPIカード          │  ④ 期日遅れ一覧                    │
│  ・総案件数            │  （テーブル: 担当/品番/品名/期日/   │
│  ・期日遅れ件数        │    ステータス/遅れ日数）             │
│  ・型UP待ち件数        │                                    │
├──────────────────────┴──────────────────────────────────┤
│  ⑤ データ検索テーブル（全データ、スライサーでフィルター）     │
│  フィルター: 担当 / ステータス / 客先 / 完了除外              │
└─────────────────────────────────────────────────────────┘
```

## STEP 1: SharePoint Excelに接続

1. Power BI Desktop を開く
2. 「データを取得」→「Web」を選択
3. URLに以下を入力:
   ```
   https://shinnihon.sharepoint.com/sites/msteams_0c471c/_layouts/15/Doc.aspx?sourcedoc={B0E3AA1F-ABE0-4087-81C7-342E3E3213E2}&action=embedview
   ```
   ※ うまくいかない場合は「SharePoint フォルダー」コネクタを使用:
   - サイトURL: `https://shinnihon.sharepoint.com/sites/msteams_0c471c`
   - ファイルを選択 → 「【技術】更新型進捗管理一覧表.xlsx」
4. 「更新型」シートを選択して「読み込み」

### 接続がうまくいかない場合の代替手段
- 「データを取得」→「Excel ブック」→ ダウンロード済みファイルを選択
- あとからSharePoint接続に切り替え可能

## STEP 2: データ整形（Power Query）

「データの変換」をクリックして Power Query エディターを開く。

### 2-1. ヘッダー設定
- 1行目はメモ行なので削除（「行の削除」→「上位の行の削除」→ 1行）
- 「1行目をヘッダーとして使用」をクリック

### 2-2. 列名の整理（わかりやすくリネーム）
| 元の列 | リネーム後 |
|---|---|
| 列F（担当） | 担当 |
| 列M（ステータス） | ステータス |
| 列O（期日） | 期日 |
| 列G（品番） | 品番 |
| 列H（品名） | 品名 |
| 列I（更新型） | 更新型 |
| 列D（客先） | 客先 |
| 列A（管理番号） | 管理番号 |
| 列N（現状進捗と課題） | 進捗メモ |
| 列f（完了タグ） | 完了タグ |

### 2-3. データ型の設定
- 期日 → 「日付」型に変更
- 管理番号 → 「整数」型
- 完了タグ → 「整数」型

### 2-4. フィルター（空行・不要行の除外）
- 管理番号が null または空白の行を除外
- 「閉じて適用」

## STEP 3: DAX計算式（メジャー）

「新しいメジャー」から以下を追加。

### 3-1. 総案件数（未完了のみ）
```dax
総案件数 =
COUNTROWS(
    FILTER('更新型', '更新型'[完了タグ] = 0 || ISBLANK('更新型'[完了タグ]))
)
```

### 3-2. 期日遅れ件数
```dax
期日遅れ件数 =
COUNTROWS(
    FILTER(
        '更新型',
        '更新型'[期日] < TODAY()
        && ('更新型'[完了タグ] = 0 || ISBLANK('更新型'[完了タグ]))
        && NOT(ISBLANK('更新型'[期日]))
        && '更新型'[ステータス] <> "引き渡し準備済"
        && '更新型'[ステータス] <> "合格済/予備ピン手配中"
    )
)
```

### 3-3. 遅れ日数（計算列）
「新しい列」から追加:
```dax
遅れ日数 =
IF(
    NOT(ISBLANK('更新型'[期日])) && '更新型'[期日] < TODAY(),
    DATEDIFF('更新型'[期日], TODAY(), DAY),
    BLANK()
)
```

### 3-4. 型UP待ち件数
```dax
型UP待ち件数 =
COUNTROWS(
    FILTER('更新型', '更新型'[ステータス] = "型UP待ち")
)
```

### 3-5. ステータス進捗率
```dax
完了率 =
DIVIDE(
    COUNTROWS(FILTER('更新型', '更新型'[ステータス] = "引き渡し準備済" || '更新型'[ステータス] = "合格済/予備ピン手配中")),
    COUNTROWS('更新型'),
    0
)
```

## STEP 4: ビジュアル配置

### ① 担当別案件数（円グラフ）
- ビジュアル: 円グラフ
- 凡例: 担当
- 値: 管理番号のカウント
- フィルター: 完了タグ ≠ 1

### ② ステータス別積み上げ棒グラフ
- ビジュアル: 積み上げ横棒グラフ
- 軸: 担当
- 凡例: ステータス
- 値: 管理番号のカウント
- 並べ替え: 件数降順

### ③ KPIカード（3枚）
- カード1: 総案件数（メジャー）
- カード2: 期日遅れ件数（メジャー）→ 赤色
- カード3: 型UP待ち件数（メジャー）

### ④ 期日遅れ一覧テーブル
- ビジュアル: テーブル
- 列: 担当 / 品番 / 品名 / 客先 / ステータス / 期日 / 遅れ日数
- フィルター: 遅れ日数 > 0
- 条件付き書式: 遅れ日数が大きいほど赤く

### ⑤ データ検索テーブル
- ビジュアル: テーブル
- 列: 管理番号 / 担当 / 品番 / 品名 / 更新型 / ステータス / 期日 / 進捗メモ

### スライサー（フィルター）
- 担当（ドロップダウン）
- ステータス（ドロップダウン）
- 客先（ドロップダウン）
- 完了除外トグル（完了タグ = 0）

---

## STEP 5: 自動更新設定

### Power BI Serviceに発行する場合
1. 「発行」→ ワークスペースを選択
2. データセットの設定 → 「スケジュールされた更新」
3. SharePointの資格情報を設定
4. 更新頻度: 1日1回（朝8:00など）

### ローカル運用の場合
- Power BI Desktop で「更新」ボタンを押すだけ
- SharePoint上のExcelが更新されていれば最新データが反映される
