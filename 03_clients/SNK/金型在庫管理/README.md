## すぐに試す（デモモード）

1. `金型在庫管理.html` をブラウザで開く
2. ログインなしで全機能が使える（データはブラウザのローカルストレージに保存）
3. サンプルデータが5件入っている

## 本番環境への接続手順

### Step 1: Azure ADにアプリを登録

1. [Azure Portal](https://portal.azure.com) → Azure Active Directory → アプリの登録
2. 「新規登録」をクリック
3. 名前: `金型在庫管理アプリ`
4. リダイレクトURI: `SPA` を選択し、アプリのURL（SharePointページのURL）を入力
5. 「APIのアクセス許可」→ 以下を追加:
   - `Sites.ReadWrite.All`（委任）
   - `User.Read`（委任）
6. 「管理者の同意を与える」をクリック
7. クライアントIDをメモ

### Step 2: SharePointリストを作成

`SharePointリスト定義.md` に従ってリストを作成する。

- 既存の「更新型一覧表」にカラムを追加
- 「ラック在庫」リストを新規作成
- ラック在庫の初期データ（全番地を空で登録）を投入

### Step 3: アプリの設定を変更

`金型在庫管理.html` の `CONFIG` セクションを編集:

```javascript
const CONFIG = {
    clientId: 'ここにAzure ADのクライアントIDを貼り付け',
    authority: 'https://login.microsoftonline.com/ここにテナントID',
    sharepointHost: 'yourcompany.sharepoint.com',
    sharepointSitePath: '/sites/your-site',
    listA_name: '更新型一覧表',  // 実際のリスト名に合わせる
    listB_name: 'ラック在庫',    // 実際のリスト名に合わせる
    racks: {
        // 実際のラック構成に合わせて番地を設定
        'D1': generateAddresses(5, 6),  // 5行 x 6列 = 30番地
        // ...
    }
};
```

### Step 4: SharePointに配置

- SharePointサイトの「サイトのページ」に新しいページを作成
- 「埋め込み」Webパーツを追加し、HTMLコードを貼り付け
- または、SharePointのドキュメントライブラリにHTMLを直接アップロード

## 機能一覧

| 機能 | 説明 |
|---|---|
| QRスキャン | カメラで金型/ラックのQRコードを読み取り |
| 入庫登録 | 金型をラック番地に登録。重複・上書き防止あり |
| 出庫 | 金型をラックから取り出し。両リストを更新 |
| 移動 | 別の番地へ移動（旧番地クリア → 新番地登録） |
| 検索 | 金型Noで現在の保管場所を表示 |
| ダッシュボード | 全ラックの使用率をグラフ・タイル表示 |
| 棚卸モード | ラック単位で中身を1つずつ確認 |
| オフライン対応 | 電波が弱い場所でもローカル保存→復帰後同期 |

## バリデーション（エラー防止）

- 既に金型が入っている番地への入庫 → 警告表示
- 同じ金型が別の場所に既にある場合 → 警告表示
- 存在しない番地の指定 → エラー表示

## QRコードのフォーマット

| 用途 | フォーマット | 例 |
|---|---|---|
| 金型用QR | `MOLD:金型No` | `MOLD:K-12345` |
| ラック番地用QR | `RACK:ラック-番地` | `RACK:D1-A3` |

## カスタマイズのヒント

### ラック構成の変更
`CONFIG.racks` を変更するだけで、ラック数・番地数を自由に変更できる。

### 権限管理
Graph APIの委任アクセス許可を使っているので、SharePoint側のリスト権限がそのまま適用される。
閲覧のみ・編集可能の制御はSharePointリストの権限設定で行う。

### QRコード印刷
金型やラック番地のQRコードは、Excelのマクロやフリーツールで一括生成すると効率的。
フォーマットは `MOLD:K-12345` または `RACK:D1-A3` に統一する。

### オフライン対応の強化
現在はローカルストレージに一時保存する方式。
Service Workerを追加すれば、アプリ自体もオフラインで開けるようになる（PWA化）。
