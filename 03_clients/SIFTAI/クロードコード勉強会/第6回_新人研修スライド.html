🔒 ガードレールの考え方

2つの防御レイヤーでAIを制御する

🚧

物理的な制限

ファイルの読み取り権限を剥奪
（permissions.deny）

道路の赤コーン
物理的に通れない

🧠

行動の制限

AIの判断基準にルールを組み込む
（CLAUDE.md / Rules）

運転者のモラル教育
自分で判断して止まる

💡 赤コーン（物理制限）で通れなくして、モラル教育（行動制限）で自ら止まる。
この2重の防御がガードレール。

06 / 10

🔧 設定方法：permissions.deny

.claude/settings.json に追記するだけ

// .claude/settings.json
{
  "permissions": {
    "deny": [
      "Read(**/*.env)",
      "Read(**/*.key)",
      "Read(**/*.pem)",
      "Read(**/*.crt)",
      // ↓ プロジェクト固有のパターンを追加
      "Read(09_system/config/*_token*)",
      "Read(09_system/config/google_oauth_*)"
    ]
  }
}
⚠ 注意点

汎用パターン（.env, .key等）だけでは不十分。
自分のプロジェクトの機密ファイルを棚卸しして、固有のパターンを追加すること。

📊 .gitignore vs permissions.deny

「.gitignoreがあるから安心」は半分だけ正解

.gitignore permissions.deny
守る対象 GitHubへのアップロード AIからの読み取り
守るタイミング git add / git push 時 AIがファイルを参照する時
守れない場面 ローカルでAIが読むのは防げない Gitへのpushは防げない

👉 両方設定して初めて完全防御
.gitignore = 対人間の防御 ／ permissions.deny = 対AIの防御
