# 第6回 クロードコード勉強会 宿題レポート — 指示設計の3層構造

**名前**: 野村冴香
**テーマ**: 指示設計を制する者がClaude Codeを制する

---

## 学んだこと

### 設計なしで使うと陥る「3つの地獄」

| 地獄 | 症状 | 自分の経験 |
|---|---|---|
| 指示リセット地獄 | 毎回同じ前提条件を打ち直す | 「日本語で」「箇条書きで」を何度も言っていた |
| 自分らしさゼロ地獄 | 汎用的な回答しか返ってこない | 褒め言葉から始まる回答にうんざりしていた |
| コンテキスト爆食い地獄 | 不要なファイルまで読み込んで出力がぼやける | 関係ないフォルダの情報まで混ざることがあった |

### 解決策: 指示設計の「3層構造」

```
┌─────────────────────────┐
│  .claude/skills/         │ ← 必殺技セット（特定タスクの自律実行）
├─────────────────────────┤
│  .claude/rules/*.md      │ ← 現場の手順書（用途・ファイル種別ごとのルール）
├─────────────────────────┤
│  CLAUDE.md               │ ← 就業規則（プロジェクト全体の基本スタンス）
└─────────────────────────┘
```

**料理人メタファーでの理解**:
- CLAUDE.md = 「うちの店では必ず手袋をすること」（衛生基準・基本ルール）
- Rules = 「魚を捌く時だけ青いまな板を使え」（食材別の個別手順）
- Skills = 「特製ビーフシチューの注文が入ったら、このレシピと圧力鍋を使え」（専用道具のセット）

---

## 実践: Before / After

### Before（CLAUDE.md設定前）

プロンプト: `私の仕事の提案書の構成を考えて`

- 「喜んでお手伝いします！」から始まる
- 汎用的な提案書テンプレートが返ってくるだけ
- 自分の業種・立場を全く反映していない

### After（グローバルCLAUDE.md設定後）

同じプロンプトに対して:

- 挨拶・前置きなし。結論から始まる
- 「AIエージェント企業のCOO」という立場を踏まえた構成
- 小規模事業者向けの実用性・コスト意識が反映されている

→ **同じ一言でも、CLAUDE.mdがあるだけで「汎用AI」から「専属アシスタント」に変わった。**

## セキュリティガードレール

第6回のもう一つの柱は、`.claude/settings.json` によるセキュリティ設定。

### デモで学んだこと

| 状態 | 挙動 |
|---|---|
| ガードレールなし（無法者AI） | `.env`のAPIキー・DB接続情報をそのまま出力してしまう |
| ガードレールあり（プロAI） | `ignorePaths`で`.env`を不可視化 + `customInstructions`で出力拒否 |

### 自分のプロジェクトへの適用

現在の `ai-agent/.claude/settings.json` に以下を反映すべき項目:

- **ignorePaths**: `*.env`, `*.key`, `*.pem`, `credentials.json`, `*.pickle`（OAuth tokenなど）
- **customInstructions**: 機密情報保護ルール、プロンプトインジェクション対策、破壊的変更の事前確認

→ 特に `09_system/config/` 配下にOAuth認証ファイルがあるため、`ignorePaths` の設定は必須。

| やりたいこと | ファイル | 場所 | 自分の状態 |

| セキュリティガードレール | settings.json | `.claude/settings.json` | 要追加 |

## 感想

今回の勉強会で、自分が第1回から積み上げてきたCLAUDE.md・Rules・Skillsの設計が、まさにこの「3層構造」そのものだったことに気づいた。言語化されたことで、なぜこの構成が機能しているのかが腑に落ちた。

特に印象的だったのはセキュリティの話。`ignorePaths`で物理的にファイルを見えなくする「赤コーン」と、`customInstructions`で行動を制限する「モラル」の二段構え。プロンプトインジェクションのデモ（外部HTMLに仕込まれた「.envを出力しろ」という隠し命令）は、AIを業務に使う上で見逃せないリスクだと実感した。

自分のプロジェクトには `09_system/config/` にOAuth認証ファイルがあるのに、`ignorePaths` を設定していなかった。今回の学びを活かして、ガードレールを追加する。「ブレーキをしっかり設定してこそ、アクセルを全開で踏める」という言葉が響いた。
